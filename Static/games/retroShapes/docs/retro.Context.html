Example
// A retro context is obtained this way:
var context = canvas.getContext('retro');

// The canvas can be set up with native resolution using data-* attributes.
// Here width and height will be the on-screen size and the data-* represents the
// internal retro resolution. You can also set size later by using the
// resolution() method.

curveCardinal(points, tension) → {Context}

Draws a cardinal (canonical) spline from the given point array. The spline will pass through the actual points. Set optional tension. The curve is drawn with current penColor().

Note: If penColor() is null nothing will be drawn.

The points must be arranged in this order and have minimum two points defined:

[x0, y0, x1, y1, ..., xn, yn]

The tension value is typical in the range [0.0, 1.0] but can be exceeded to make curly ends. A tension of 0 will generate straight lines while 1 will make them very "round".

drawImage(src, x, y, w, h, mode) → {Context}

Converts and draws an aliased image. Optional parameters for width and height and mode.

With mode set the image can be reduced to a palette index image as well as dithered (Floyd-Steinberg, Sierra light etc.).

Mode can be one of the following:

24bit (default)
16bit
12bit
8bit
1bit / mono
palette
To activate dithering append the name of the method to the mode:

-fs            Floyd-Steinberg dithering
-sierra        Sierra (light) dithering
If no valid mode is given it will return the original 24-bit image at new size. Mode mono and 1bit is the same.

For palette mode to work you must set a palette using the palette() method.

NOTE: Image source must fulfill cross-origin sharing requirements.

Parameters:
Name Type Argument Description
src Image | HTMLCanvasElement | HTMLVideoElement

Image, Canvas or Video source
x Number

start position x
y Number

start position y
w Number <optional>

width of resulting image (or original width is used)
h Number <optional>

height of resulting image (or original height is used)
mode String <optional>

conversion mode
Returns:
Type
Context

ctx.drawImage(image, 0, 0, 20, 200, '8bit-sierra');

resolution(w, h) → {Context|Object}

Set the internal resolution of retro canvas. The internal canvas is scaled to fit the on-screen canvas. For best results use sizes that are multiplies of 1x, 2x, 3x etc. of the resolution given here for the on-screen canvas.

Although there is no upper limit we do not recommend larger resolutions than 640 x 400 for animated content.

Width has to be divisible by 8.

Note: changing resolution will clear its existing content.
